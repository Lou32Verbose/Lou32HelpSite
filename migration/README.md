# Legacy Docs Migration

`migration/legacy-docs-manifest.csv` is the working inventory for every source
file under `docs/`.

Columns:

- `source_file`: legacy filename in `docs/`
- `disposition`: `migrate`, `merge`, `split`, or `archive`
- `canonical_cluster`: shared destination cluster or inventory grouping
- `destination_topic`: target topic path when the file is migrated or grouped
- `destination_slug`: canonical content slug when one exists
- `page_type`: target page type for migrated content
- `status`: planned document status for the destination page
- `review_notes`: migration rationale or cleanup notes

The manifest is generated by `scripts/generate-legacy-docs-manifest.ps1`.
Re-run that script whenever new legacy files are added or migration rules are
adjusted.
