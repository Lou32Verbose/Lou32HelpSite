1. The Pipeline

The pipeline operator | sends the output of one command into the next command.


Get-Service | Select-Object Name, Status

This means:

  1. Get all services.
  2. Send them to Select-Object.
  3. Display only the Name and Status properties.

Multi-line formatting is often easier to read:


Get-Service |
    Select-Object Name, Status

2. Inspect Object Properties

Before filtering or selecting, inspect what properties are available.


Get-Service | Get-Member

Shorter alias:


Get-Service | gm

Example with processes:


Get-Process | Get-Member

To see all properties for one object:


Get-Service |
    Select-Object -First 1 *

Or format the first object as a list:


Get-Service |
    Select-Object -First 1 |
    Format-List *

3. Filtering with Where-Object

Use Where-Object to keep only objects that match a condition.

Long form:


Get-Service |
    Where-Object { $_.Status -eq 'Running' }

Short form:


Get-Service |
    Where-Object Status -eq 'Running'

$_ means "the current object in the pipeline."

4. Common Comparison Operators

OperatorMeaning
-eqEquals
-neNot equals
-gtGreater than
-geGreater than or equal
-ltLess than
-leLess than or equal
-likeWildcard match
-notlikeWildcard does not match
-matchRegex match
-notmatchRegex does not match
-inValue is in a collection
-notinValue is not in a collection
-containsCollection contains value
-notcontainsCollection does not contain value

Example:


Get-Process |
    Where-Object CPU -gt 10

5. Wildcard Filtering with -like

Use -like when you want simple wildcard matching.

PatternMeaning
abc*Starts with abc
*abcEnds with abc
*abc*Contains abc
a?cMatches abc, axc, etc.

Example: services starting with Win


Get-Service |
    Where-Object Name -like 'Win*'

Example: packages with IDs starting with MSIX/


Get-WingetPackage |
    Where-Object Id -like 'MSIX/*'

Example: processes containing edge


Get-Process |
    Where-Object ProcessName -like '*edge*'

6. Regex Filtering with -match

Use -match for regular expressions.

Example: names starting with Win


Get-Service |
    Where-Object Name -match '^Win'

Example: names ending with Svc


Get-Service |
    Where-Object Name -match 'Svc$'

Example: package IDs starting with MSIX/


Get-WingetPackage |
    Where-Object Id -match '^MSIX/'

Regex anchors:

RegexMeaning
^Start of string
$End of string
.Any single character
.*Any number of characters
\dDigit
\wWord character

7. Selecting Properties with Select-Object

Use Select-Object to choose which properties to display.


Get-Service |
    Select-Object Name, DisplayName, Status

Example:


Get-Process |
    Select-Object Name, Id, CPU, WorkingSet64

Select the first 10 objects:


Get-Process |
    Select-Object -First 10

Select the last 10 objects:


Get-Process |
    Select-Object -Last 10

Skip the first 5 objects:


Get-Process |
    Select-Object -Skip 5

8. Expanding a Single Property

By default, Select-Object Name returns objects with a Name property.


Get-Service |
    Select-Object Name

Output shape:


Name
----
AarSvc
AdobeARMservice
AppIDSvc

If you only want the raw string values, use -ExpandProperty.


Get-Service |
    Select-Object -ExpandProperty Name

Output shape:


AarSvc
AdobeARMservice
AppIDSvc

This is useful when passing property values to another command.

Example:


Get-WingetPackage |
    Where-Object Id -like 'MSIX/*' |
    Select-Object -ExpandProperty Id

9. Sorting with Sort-Object

Sort objects by a property.


Get-Service |
    Sort-Object Name

Sort descending:


Get-Process |
    Sort-Object CPU -Descending

Top 10 processes by CPU:


Get-Process |
    Sort-Object CPU -Descending |
    Select-Object -First 10 Name, Id, CPU

Sort by multiple properties:


Get-Service |
    Sort-Object Status, Name

10. Grouping with Group-Object

Use Group-Object to count objects by property value.

Group services by status:


Get-Service |
    Group-Object Status

Example output:


Count Name                      Group
----- ----                      -----
  142 Running                   {...}
  120 Stopped                   {...}

Group processes by company:


Get-Process |
    Group-Object Company

Group installed packages by source:


Get-WingetPackage |
    Group-Object Source

Sort groups by count:


Get-Service |
    Group-Object Status |
    Sort-Object Count -Descending

11. Formatting Output

PowerShell has formatting commands such as:

CommandPurpose
Format-TableDisplay as a table
Format-ListDisplay as a list
Format-WideDisplay wide columns

Example:


Get-Service |
    Select-Object Name, Status |
    Format-Table -AutoSize

Format as list:


Get-Service |
    Select-Object -First 5 Name, DisplayName, Status |
    Format-List

Important: formatting commands should usually be last.

Good:


Get-Service |
    Where-Object Status -eq 'Running' |
    Select-Object Name, Status |
    Format-Table -AutoSize

Avoid this:


Get-Service |
    Format-Table |
    Where-Object Status -eq 'Running'

Once data is formatted, it is no longer as useful for further object-based processing.

12. Exporting to CSV

Use Export-Csv to save object data.


Get-Service |
    Select-Object Name, DisplayName, Status |
    Export-Csv "$env:USERPROFILE\Desktop\Services.csv" -NoTypeInformation

Export running services:


Get-Service |
    Where-Object Status -eq 'Running' |
    Select-Object Name, DisplayName, Status |
    Export-Csv "$env:USERPROFILE\Desktop\RunningServices.csv" -NoTypeInformation

Open the CSV afterward:


Invoke-Item "$env:USERPROFILE\Desktop\RunningServices.csv"

13. Exporting to JSON

Use ConvertTo-Json for structured data.


Get-Service |
    Select-Object Name, DisplayName, Status |
    ConvertTo-Json

Save JSON to a file:


Get-Service |
    Select-Object Name, DisplayName, Status |
    ConvertTo-Json |
    Set-Content "$env:USERPROFILE\Desktop\Services.json"

For nested objects, increase depth:


Get-Process |
    Select-Object -First 5 * |
    ConvertTo-Json -Depth 5

14. Calculated Properties

Calculated properties let you create custom output columns.

Example: show process memory in MB.


Get-Process |
    Select-Object Name, Id,
        @{
            Name = 'MemoryMB'
            Expression = { [math]::Round($_.WorkingSet64 / 1MB, 2) }
        }

Example: disk size and free space in GB.


Get-CimInstance Win32_LogicalDisk |
    Where-Object DriveType -eq 3 |
    Select-Object DeviceID, VolumeName,
        @{
            Name = 'SizeGB'
            Expression = { [math]::Round($_.Size / 1GB, 2) }
        },
        @{
            Name = 'FreeGB'
            Expression = { [math]::Round($_.FreeSpace / 1GB, 2) }
        }

Structure:


@{
    Name = 'NewColumnName'
    Expression = { value calculation here }
}

15. Combining Multiple Conditions

Use -and, -or, and -not.

Running services with names starting with Win:


Get-Service |
    Where-Object {
        $_.Status -eq 'Running' -and
        $_.Name -like 'Win*'
    }

Stopped services whose names contain update:


Get-Service |
    Where-Object {
        $_.Status -eq 'Stopped' -and
        $_.Name -like '*update*'
    }

Processes using high memory or high CPU:


Get-Process |
    Where-Object {
        $_.WorkingSet64 -gt 500MB -or
        $_.CPU -gt 100
    }

Negation:


Get-Service |
    Where-Object {
        $_.Status -ne 'Running'
    }

Or:


Get-Service |
    Where-Object {
        -not ($_.Status -eq 'Running')
    }

16. Finding Top Results

Top 10 processes by memory:


Get-Process |
    Sort-Object WorkingSet64 -Descending |
    Select-Object -First 10 Name, Id,
        @{
            Name = 'MemoryMB'
            Expression = { [math]::Round($_.WorkingSet64 / 1MB, 2) }
        }

Top 20 largest files in a folder:


Get-ChildItem "C:\Temp" -File -Recurse |
    Sort-Object Length -Descending |
    Select-Object -First 20 FullName,
        @{
            Name = 'SizeMB'
            Expression = { [math]::Round($_.Length / 1MB, 2) }
        }

17. Searching Files

Find files by extension:


Get-ChildItem "C:\Temp" -Filter *.log -Recurse

Find large files:


Get-ChildItem "C:\Temp" -File -Recurse |
    Where-Object Length -gt 100MB

Find recently modified files:


Get-ChildItem "C:\Temp" -File -Recurse |
    Where-Object LastWriteTime -gt (Get-Date).AddDays(-7)

Find files containing text:


Select-String -Path "C:\Temp\*.log" -Pattern "error"

Recursive text search:


Get-ChildItem "C:\Temp" -Filter *.log -Recurse |
    Select-String -Pattern "error"

18. Querying Services

All services:


Get-Service

Running services:


Get-Service |
    Where-Object Status -eq 'Running'

Stopped services:


Get-Service |
    Where-Object Status -eq 'Stopped'

Services matching a name:


Get-Service |
    Where-Object Name -like '*update*'

Services sorted by status then name:


Get-Service |
    Sort-Object Status, Name |
    Select-Object Name, DisplayName, Status

Services with startup type:


Get-CimInstance Win32_Service |
    Select-Object Name, DisplayName, State, StartMode

Automatic services that are not running:


Get-CimInstance Win32_Service |
    Where-Object {
        $_.StartMode -eq 'Auto' -and
        $_.State -ne 'Running'
    } |
    Select-Object Name, DisplayName, State, StartMode

19. Querying Processes

All processes:


Get-Process

Processes matching name:


Get-Process |
    Where-Object ProcessName -like '*edge*'

High CPU processes:


Get-Process |
    Where-Object CPU -gt 10 |
    Sort-Object CPU -Descending |
    Select-Object Name, Id, CPU

High memory processes:


Get-Process |
    Sort-Object WorkingSet64 -Descending |
    Select-Object -First 15 Name, Id,
        @{
            Name = 'MemoryMB'
            Expression = { [math]::Round($_.WorkingSet64 / 1MB, 2) }
        }

20. Querying Network Connections

Established TCP connections:


Get-NetTCPConnection |
    Where-Object State -eq 'Established'

Listening ports:


Get-NetTCPConnection |
    Where-Object State -eq 'Listen' |
    Select-Object LocalAddress, LocalPort, OwningProcess

Listening ports with process name:


Get-NetTCPConnection |
    Where-Object State -eq 'Listen' |
    Select-Object LocalAddress, LocalPort, OwningProcess,
        @{
            Name = 'ProcessName'
            Expression = {
                (Get-Process -Id $_.OwningProcess -ErrorAction SilentlyContinue).Name
            }
        }

Connections to a specific remote port:


Get-NetTCPConnection |
    Where-Object RemotePort -eq 443

22. The Most Reusable Pattern

Use this as a template:


Command |
    Where-Object Property -like 'pattern' |
    Select-Object Property1, Property2 |
    Sort-Object Property1

Example:


Get-WingetPackage |
    Where-Object Id -like 'MSIX/*' |
    Select-Object Name, Id, Version |
    Sort-Object Id

Another example:


Get-Service |
    Where-Object Name -like 'Win*' |
    Select-Object Name, DisplayName, Status |
    Sort-Object Name

24. Practical Troubleshooting Tips

If a property does not work, inspect the object


Command |
    Get-Member

If output is cut off, use Format-List


Command |
    Select-Object -First 1 |
    Format-List *

If you only need values, use -ExpandProperty


Command |
    Select-Object -ExpandProperty Name

Put formatting commands last

Good:


Get-Service |
    Where-Object Status -eq 'Running' |
    Format-Table -AutoSize

Avoid:


Get-Service |
    Format-Table -AutoSize |
    Where-Object Status -eq 'Running'

Prefer objects over text

PowerShell is strongest when you keep data as objects until the final output step.

Good:


Get-Service |
    Where-Object Status -eq 'Running' |
    Select-Object Name, Status

Less ideal:


Get-Service |
    Out-String |
    Select-String "Running"

Summary

Most PowerShell queries are built from a few core commands:

CommandPurpose
Where-ObjectFilter objects
Select-ObjectChoose properties
Sort-ObjectSort results
Group-ObjectGroup and count
Format-TableDisplay table output
Format-ListDisplay detailed output
Export-CsvSave structured results

Core pattern:


Get-Something |
    Where-Object SomeProperty -like 'SomePattern' |
    Select-Object Property1, Property2 |
    Sort-Object Property1
